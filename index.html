Project Vibl

A Feedback tool for Web developers to improve the accessibility of applications they build

Welcome to Our Project Page!

This webpage is for our project in CS770 Human-Computer Interaction. The goal of the project is to build a tool that helps check the accessibility of a webpage and encourages web developers build a more accessible website.

Motivation

Although there are different guidelines and laws to ensure the accessibility of a websites, there are still plenty of websites not accessible enough for visually impaired users. According to this website, 97 % of websites does not provide minimum levels of accessibility. We think it is important to help increase the rate of accessibility of all websites. Therefore, we decide to build a tool that helps web developers add accessibility to their websites.

Research Question

  • What are the current challenges for developers to make applications accessible?
  • How can we design effective tool to help developers make websites accessible?
  • What is the effect of Vibl on accessibility during web developers’ designing process?

    • Related Work

    There are plenty of reasons that developers do not design for accessibility. Previous research papers show that …

  • Developers are lacking of awareness of the need of different users
  • Population of potential users with disability is small
  • It is hard to incorporate all special needs for different users

    • Design of Our Tool

  • Functions of the tool
      1. Warning message about which accessibility guideline your website does not pass.
      2. Encouragement message about which accessibility guidelines are successfully included in your website.
  • Gamification
      1. Give you the information that how many other websites successfully pass the test that your website does not.
      2. Give encouragement message that how many test your website successfully pass.
  • Learning
      1. Give you a guideline a day to help you learn more about web accessibility.

    • Our Hypothesis

    We assume by using our tool, web developers can use less extra effort to incorporate accessibility in their websites and they can learn more knowledge about accessibility rules during the process.

    Design of Our Study

  • Interview web developers to know the current challenge of adding web accessibility.
  • Design tools based on the interviews.
  • Evaluation of effect of the tools on accessibility.

    • Result

    We can break our project into 2 parts: before using the tool and after using the tool.
